## 一、项目背景

### 1.1 行业背景
随着共享经济的发展和用户出行需求的多样化，租车服务已成为城市出行的重要组成部分。根据艾瑞咨询报告显示，小程序已成为国内租车预定的第二大渠道，占比高达35.5%，仅次于App。支付宝平台租车交易占据全国租车市场75%的份额，成为租车行业数字化转型的核心阵地。

### 1.2 竞品分析：九号租车小程序
九号租车是九号公司于2023年12月推出的官方租车服务小程序，主要特点：

**业务模式**：
- 线上线下结合（O2O）：利用全国30多个城市的实体门店作为服务网点
- 轻资产运营：提供平台和技术支持，车辆供应和租赁流程由各门店运营
- 低门槛体验：面向学生、旅行者及短期用车群体

**核心功能**：
- 多种租期选择：日租、周租、月租、季租
- 多款车型：九号A+、BMAX、F系列、Q90等智能电动车
- 智能化服务：实时车辆信息查询、一键导航、线上订单管理
- 便捷流程：在线预约、门店取车、续租还车

**目标用户**：
- 旅行者：短期出行需求
- 学生群体：低成本体验
- 城市通勤者：灵活替代公共交通

## 二、项目定位与目标

### 2.1 项目定位
古月租车小程序定位为**专业汽车租赁服务平台**，依托支付宝生态，为用户提供便捷、高效、安全的租车服务，覆盖城市代步、商务出行、旅游自驾等多种场景。

### 2.2 核心价值
- **用户价值**：降低用车门槛，提供灵活的出行解决方案
- **商家价值**：数字化运营工具，提升车辆周转率和运营效率
- **平台价值**：构建租车服务生态，沉淀用户数据和信用体系

### 2.3 目标用户群体
1. **商务出行人群**：企业出差、会议用车需求
2. **旅游自驾群体**：短期旅游、周末出游需求
3. **城市通勤族**：日常通勤替代方案
4. **学生群体**：低成本用车体验
5. **临时用车需求**：车辆维修、临时应急等场景

### 3.2 消息推送模块（新增）

> **补充说明**：原文档提到"消息通知"但未明确推送渠道和机制，本节完整定义消息推送体系。

#### 3.2.1 推送渠道策略

| 渠道 | 用途 | 触达方式 | 适用场景 |
|------|------|----------|----------|
| **支付宝订阅消息** | 业务通知核心渠道 | 小程序内服务通知卡片 | 取车提醒、还车提醒、订单状态变更、支付结果 |
| **阿里云短信** | 高优先级/紧急通知 | 手机短信 | 违章通知、超时还车预警、安全警告、验证码 |
| **支付宝小程序内消息** | 非实时通知 | 小程序内消息中心入口 | 优惠活动、系统公告、评价回复 |
| **App内弹窗**（预留） | 重要提示 | 打开小程序时弹出 | 新功能引导、紧急公告 |

**渠道优先级规则：**
- 订单状态变更 → 支付宝订阅消息 + 短信（双重触达）
- 营销活动 → 仅小程序内消息
- 安全/合规相关 → 短信为主 + 订阅消息为辅

#### 3.2.2 消息场景与模板定义

**业务类通知（支付宝订阅消息）：**
| 场景 | 消息标题 | 消息内容 | 发送时机 | 跳转页面 |

| 支付成功 | 支付成功确认 | 已支付{amount}元，{vehicle}租赁订单支付成功！请在取车时间到店取车。 | 支付回调成功后 | 订单详情页 |
| 取车提醒 | 取车时间提醒 | 您预订的{vehicle}已准备好，请于{pickupTime}到{storeName}取车，订单号：{orderNo}。 | 取车前2小时+30分钟各一次 | 订单详情页 |
| 还车提醒 | 还车时间提醒 | 您的租车订单{orderNo}将于{returnTime}到期，请及时归还车辆到指定门店。 | 还车前2小时+到期时各一次 | 订单详情页 |

#### 3.2.3 推送技术实现

```json
{
  "推送架构": {
    "消息队列": "RabbitMQ/Kafka - 异步解耦，削峰填谷",
    "定时任务": "XXL-JOB / Spring Scheduler - 定时扫描待发送任务",
    "重试机制": "指数退避重试(1s→2s→4s→8s→16s→32s)，最多3次",
    "降级策略": "支付宝订阅消息失败时降级为短信发送"
  }
}
```

**消息发送流程：**
```
业务事件产生 → 写入消息表(status=待发送) 
→ MQ异步消费 → 调用对应渠道API
→ 更新消息状态(成功/失败) 
→ 失败则进入重试队列
→ 重试耗尽转人工处理
```

#### 3.3.3 工单状态机

```
待处理 → 处理中 → 待用户确认 → 已完成 → 已关闭
                ↘ 已转派 ↗
                ↘ 已拒绝(不合理需求)
```

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 10 | 待处理 | 工单已提交，等待客服接单 |
| 20 | 处理中 | 客服正在处理中 |
| 30 | 待用户确认 | 方案已给出，等待用户反馈 |
| 40 | 已完成 | 问题已解决，用户满意 |
| 50 | 已关闭 | 工单结束（含用户主动关闭） |
| 60 | 已转派 | 转给其他部门/人员处理 |
| 70 | 已拒绝 | 不合理诉求，已关闭 |

### 3.9 异常处理流程规范（新增）

> **补充说明**：原订单状态80"异常"仅简单提及，本节细化所有异常类型和处理SOP。

#### 3.9.1 异常类型定义

| 异常编码 | 异常类型 | 严重度 | 触发条件 |
|----------|----------|--------|----------|
| EX-001 | **超时还车** | 中 | 超过预定还车时间未归还车辆 |
| EX-002 | **车辆损坏** | 高 | 还车时检查发现车辆有损坏 |
| EX-003 | **违章记录** | 中 | 租期内产生交通违章 |
| EX-004 | **车辆故障** | 高 | 租用期间车辆发生故障无法正常行驶 |
| EX-005 | **事故报案** | 严重 | 租用期间发生交通事故 |
| EX-006 | **车辆丢失** | 严重 | 车辆去向不明或被盗 |
| EX-007 | **支付异常** | 中 | 支付失败、重复扣款、退款超时 |
| EX-008 | **用户纠纷** | 低 | 用户投诉服务质量、价格争议等 |
| EX-009 | **系统异常** | 中 | 数据错误、接口超时等技术问题 |
| EX-010 | **恶意行为** | 高 | 用户疑似诈骗、套取车辆等 |

#### 3.9.2 各异常处理SOP

**EX-001 超时还车处理流程：**
```
系统检测超时 → 自动发送短信提醒(每4小时)
→ 超2小时:自动按超时规则计费
→ 超4小时:短信+电话通知用户
→ 超24小时:人工介入+额外收取50元/天服务费
→ 超72小时且联系不上用户:
  → 冻结芝麻信用/扣除全额押金
  → 报警备案(车辆可能丢失)
  → 订单转EX-010处理
```

**EX-002 车辆损坏处理流程：**
```
还车检查发现损坏 → 拍照留证(多角度)
→ 评估维修费:
  ├── <500元: 用户现场赔付,快速结算
  ├── 500-5000元: 走保险理赔流程,用户承担免赔额部分
  ├── >5000元: 报保险+定损,用户承担免赔额+折旧费
  └── 全损: 保险全赔+折旧补偿
→ 维修期间车辆下架
→ 维修完成后重新上架
→ 所有过程记录存档(照片/单据)
```

**EX-003 违章记录处理流程：**
```
交管局推送违章数据 → 系统创建异常工单(EX-003)
→ 通知用户(小程序消息+短信)
→ 用户15日内自行处理 → 上传凭证 → 核实 → 解冻违章押金
→ 用户逾期未处理(15天):
  → 短信+电话催促
→ 逾期30天仍未处理:
  → 平台代为处理(从违章押金扣除罚款+200元代办费)
  → 通知用户扣款明细
→ 违章押金不足:
  → 追缴差额欠款
  → 影响芝麻信用评分
  → 列入平台限制名单
```

**EX-004 车辆故障处理流程（租用中）：**
```
用户报障 → 客服记录故障现象
→ 引导用户安全停车
→ 派遣附近门店救援/拖车
→ 安排替代车辆(如有)或全额退还剩余租金
→ 故障车送修
→ 如因车辆质量问题导致用户损失:
  → 平台承担责任
  → 协商赔偿方案
```

**EX-005 事故报案处理流程：**
```
用户报事故 → 立即确认人身安全
→ 指导用户报警(122)+报保险
→ 收集事故信息(时间/地点/对方信息/现场照片)
→ 协助保险公司定损
→ 根据责任判定处理:
  ├── 用户全责: 保险赔付免赔额内,超额用户自担
  ├── 对方全责: 对方保险公司赔付
  └── 双方有责: 按比例分担
→ 事故处理完成后关闭异常
```

**EX-010 恶意行为处理流程：**
```
风控系统检测可疑行为(IP异常/虚假信息/频繁取消订单等)
→ 人工复核
→ 确认恶意:
  ├── 冻结账号
  ├── 扣除押金
  ├── 列入黑名单(设备ID+支付宝ID+手机号)
  ├── 必要时报案(涉嫌诈骗)
  └── 通报芝麻信用降分
→ 误判: 解除限制,恢复正常
```

#### 3.9.3 异常升级机制

| 当前级别 | 处理人 | 升级条件 | 升级后级别 |
|----------|--------|----------|------------|

#### 3.10.1 用户协议（《古月租车服务协议》）

协议必须包含以下核心条款：

| 条款类别 | 具体内容要点 |

| **赔偿责任** | 不同损坏程度的赔偿标准（参照10.5节） |
| **违约责任** | 超时还车、违章不处理、恶意拖欠等违约情形的处理 |
| **隐私保护** | 信息收集范围、使用目的、共享边界、用户权利 |
| **争议解决** | 协商→平台调解→诉讼（约定管辖法院） |
| **免责声明** | 不可抗力、第三方原因导致的损失免责范围 |

#### 3.10.2 隐私政策要点

| 项目 | 说明 |
|------|------|
| **收集的信息** | 支付宝授权信息(昵称/头像/实名)、手机号、驾驶证信息、身份证号、位置信息、支付信息 |
| **信息用途** | 身份认证、订单履约、支付结算、信用评估、安全风控、改善服务 |

## 五、业务流程设计

### 5.1 用户租车核心流程

```
用户登录 → 实名认证 → 驾驶证认证 → 车辆浏览 → 选择车辆 → 确认订单 
→ 在线支付 → 门店取车 → 使用车辆 → 归还车辆 → 费用结算 → 订单评价
```

#### 详细流程说明：

**第一阶段：用户注册与认证**
1. 用户通过支付宝一键授权登录
2. 系统自动获取支付宝实名认证信息
3. 用户上传驾驶证照片，系统OCR识别+人工审核
4. 审核通过后，开启租车权限

**第二阶段：车辆选择与预订**
1. 用户浏览车辆列表，筛选车型、价格、品牌等
2. 查看车辆详情、门店位置、可租时间
3. 选择取还车门店、时间
4. 系统自动计算租金、保险费、服务费
5. 确认订单信息，提交订单

**第三阶段：支付与取车**
1. 用户选择支付方式（信用免押或押金预授权）
2. 完成支付后，订单状态变更为"待取车"
3. 用户收到取车提醒通知
4. 用户到店出示订单二维码
5. 门店工作人员核对身份、交付车辆、签署协议
6. 扫码取车，订单状态变更为"进行中"

**第四阶段：用车与还车**
1. 用户使用车辆
2. 如需续租，在线申请并支付续租费用
3. 到达还车时间，用户归还车辆到指定门店
4. 门店工作人员检查车辆状况
5. 确认无异常后，计算最终费用
6. 退还多余押金（或补收费用）
7. 订单状态变更为"已完成"

**第五阶段：评价与售后**
1. 用户对车辆、服务进行评价
2. 处理违章、理赔等后续事宜
3. 用户可查看历史订单、申请发票

### 5.2 商家运营流程

```
车辆录入 → 车辆上架 → 订单处理 → 车辆交付 → 车辆回收 → 费用结算 
→ 车辆维保 → 车辆再次上架
```

### 6.5.4 商户入驻与激励机制

> **补充说明**：O2O模式的核心难点是车源获取，本节制定商户拓展计划。

#### 6.5.4.1 目标商户画像

| 商户类型 | 代表对象 | 合作价值 | 拓展难度 |
|----------|----------|----------|----------|
| **连锁租车行** | 神州/一嗨等加盟店 | 车源丰富、标准化程度高 | 高(已有合作平台) |
| **独立租车行** | 本地中小租车公司(5-50辆车) | 数量大、合作意愿强 | 中 |
| **4S店/车行** | 汽车4S店租车部门 | 车况好、品牌信任度高 | 中 |
| **个人车主** | 闲置车辆共享 | 车源长尾、灵活 | 低(风控难度高) |

**MVP阶段重点目标**：独立租车行 + 本地4S店（更容易谈合作）

#### 6.5.4.2 入驻流程

```
商户咨询 → 资质审核(营业执照+租赁许可证)
→ 合同签署 → 门店信息录入系统 
→ 车辆批量导入 → 员工账号创建 
→ 操作培训(30分钟) → 正式上线运营
```

**入驻资质清单：**
- [ ] 营业执照（经营范围含汽车租赁）
- [ ] 汽车租赁经营许可证（或当地同等资质证明）
- [ ] 法定代表人身份证
- [ ] 银行开户许可证（用于分账结算）
- [ ] 车辆行驶证 + 交强险保单
- [ ] 至少5台可租车辆

#### 6.5.4.3 商户激励机制

| 激励类型 | 具体政策 | 适用条件 |
|----------|----------|----------|
| **入驻礼包** | 免收首3个月平台服务费 | 新商户签约即享 |

| **流量扶持** | 首页推荐位 + "精选门店"标识 | 评分≥4.8且车辆数≥10台的商户 |

| **设备补贴** | 免费提供/补贴扫码枪、打印机等硬件 | 车辆数≥15台的商户 |
| **年终返点** | 年GMV的1%-3%作为年度返利 | 年交易额达标商户 |

| 收费项 | 收费标准 | 备注 |
|--------|----------|------|
| 平台服务费 | 租金的5%-8%（按阶梯） | 从用户支付中自动分账 |

## 六、数据安全与合规要求

### 6.1 数据安全要求
- **传输安全**：全站HTTPS加密，敏感数据传输加密
- **存储安全**：用户密码加密存储，敏感信息脱敏处理
- **访问控制**：基于角色的权限管理（RBAC），操作日志记录
- **数据备份**：定期备份数据库，支持数据恢复

### 6.2 合规要求
- **个人信息保护法**：明确用户隐私政策，获取用户授权
- **网络安全法**：落实网络安全等级保护要求
- **支付合规**：遵循支付宝支付规范，确保资金安全
- **租车行业规范**：遵守交通运输部租车行业管理规定

## 十、核心业务规则

### 10.1 定价规则

#### 10.1.1 基础定价模型
```
订单总费用 = 租金 + 保险费 + 服务费 + 可选增值服务费

租金 = 基础日租金 × 租赁天数 × 时段系数 × 会员折扣
```

**时段系数说明:**
| 时段类型 | 系数 | 说明 |
|---------|------|------|
| 工作日 | 1.0 | 周一至周四 |
| 周末 | 1.2 | 周五至周日 |
| 节假日 | 1.5 | 法定节假日 |
| 旺季 | 1.3-2.0 | 暑假(7-8月)、国庆等 |

**会员折扣:**
| 会员等级 | 折扣 | 成长值要求 |
|---------|------|-----------|
| 普通会员 | 无折扣 | 0-99 |
| 银卡会员 | 95折 | 100-499 |
| 金卡会员 | 90折 | 500-1999 |
| 钻石会员 | 85折 | 2000+ |

**成长值获取规则:**
- 租车1元 = 1成长值
- 邀请好友首次租车成功 = 50成长值
- 好友后续租车 = 获得好友成长值的10%

#### 10.1.2 押金规则

**车辆押金(信用免押或预授权):**
| 车型等级 | 押金金额 | 芝麻信用免押要求 |
|---------|---------|----------------|
| 经济型(10万以下) | 2000元 | ≥600分 |
| 舒适型(10-20万) | 5000元 | ≥650分 |
| 商务型(20-30万) | 8000元 | ≥680分 |
| 豪华型(30万以上) | 15000元 | ≥700分 |

**违章押金:** 统一1000元预授权,还车后30天内无违章则解冻

#### 10.1.3 保险费用
| 保险类型 | 费用/天 | 赔付范围 | 免赔额 |
|---------|---------|---------|--------|
| 基础险 | 20元 | 车损5000元以上 | 5000元 |
| 不计免赔险 | 50元 | 车损全额赔付 | 0元 |
| 盗抢险 | 30元 | 车辆被盗全赔 | 0元 |

### 10.2 订单取消与退款规则

#### 10.2.1 取消政策
| 取消时间 | 退款比例 | 扣除费用 |
|---------|---------|---------|
| 取车前48小时以上 | 100% | 无 |
| 取车前24-48小时 | 80% | 扣除20%违约金 |
| 取车前12-24小时 | 50% | 扣除50%违约金 |
| 取车前12小时内 | 0% | 不退款 |
| 已取车 | 0% | 不支持取消 |

#### 10.2.2 退款流程
```
用户申请取消 → 系统判断退款金额 → 提交退款请求 → 支付宝退款接口
→ 退款到原支付账户 → 发送退款通知 → 订单状态更新
```
- 退款时效: 1-3个工作日
- 退款方式: 原路退回

### 10.3 超时还车规则

| 超时时长 | 收费标准 | 处理方式 |
|---------|---------|---------|
| ≤2小时 | 按小时计费(日租金÷24×2) | 自动续费 |
| 2-4小时 | 按半天计费 | 自动续费 |
| >4小时 | 按整天计费 | 自动续费+短信提醒 |
| >24小时 | 按整天计费+额外服务费 | 人工介入 |

**超时服务费:** 每超时24小时额外收取50元

### 10.4 违章处理流程

```
违章产生 → 交管局推送 → 系统通知用户 → 用户15日内处理
→ 用户处理完成上传凭证 → 平台核实 → 解冻违章押金
```

**用户未处理后果:**
- 15日内未处理: 短信+电话催促
- 30日内未处理: 平台代处理,从违章押金扣除费用+200元代办费
- 扣除费用超押金: 追缴欠款,计入信用记录

### 10.5 车辆损坏处理

| 损坏程度 | 责任认定 | 赔偿标准 |
|---------|---------|---------|
| 轻微刮擦(维修费<500) | 用户承担 | 实际维修费 |

### 11.5 邀请有礼防刷机制（新增）

> **补充说明**：原邀请功能有但缺少反作弊策略。

#### 11.5.1 正常奖励规则

| 条件 | 邀请人奖励 | 被邀人奖励 |
|------|------------|------------|
| 被邀人注册成功 | +50成长值 | 新人专享礼包(优惠券×3) |
| 被邀人完成首单 | +50元现金红包(可提现) | 首单立减30元 |
| 被邀人后续每单 | 获得其消费金额10%等值积分(上限200积分/单) | 无 |

#### 11.5.2 防刷策略

| 风控项 | 检测规则 | 处理措施 |
|--------|----------|----------|
| **设备指纹重复** | 同一设备ID注册多个账号视为刷号 | 冻结关联账号、扣除已发奖励 |
| **IP地址异常** | 同一IP短时间(24h内)注册>5个账号 | 触发人工审核+图形验证码加强 |
| **手机号批量** | 批量号段/虚拟号码注册 | 直接拒绝注册 |
| **行为异常** | 注册后立即注销再注册、频繁更换设备登录 | 标记可疑账号，限制提现 |
| **虚假交易** | 邀请双方有资金往来或同一收货地址/取车模式异常 | 取消双方奖励、冻结账号 |
| **薅羊毛团伙** | 关联账号网络图谱检测（多账号互邀形成闭环） | 批量封禁、追缴已发奖励、通报芝麻信用 |
| **邀请频率限制** | 单用户每日最多邀请10人，每月最多邀请50人 | 超出后提示"邀请已达上限" |

#### 11.5.3 提现风控

- 首次提现需进行**实名认证二次验证**
- 单笔提现≥100元需**人工审核**(T+1到账)
- 单日累计提现>500元触发**反洗钱审查**
- 异常提现行为将**冻结提现功能**并要求提供资金来源证明
