**核心方法**：
```java
// OrderDAO
- insert(Order order) → 生成订单号，插入记录

- findByUserId(Integer userId, int page, int size) → 用户订单分页查询
- updateStatus(Integer orderId, String status) → 更新订单状态
- countByUserIdAndStatus(Integer userId, String status) → 统计各状态订单数

// OrderItemDAO  
- insertBatch(List<OrderItem>) → 批量插入订单项
- findByOrderId(Integer orderId) → 查询某订单的所有项目
```

### 2.3 W5.2 后端API — 订单接口

参考《后端接口文档》第五章「订单相关接口」：

| 编号 | 接口 | URL | 方法 | 优先级 |
|------|------|-----|------|-------|
| W5.2.1 | 提交租车订单 | `/api/orders` | POST | P0 |
| W5.2.2 | 获取订单列表 | `/api/orders` | GET | P0 |
| W5.2.3 | 获取订单详情 | `/api/orders/{id}` | GET | P0 |
| W5.2.4 | 取消订单 | `/api/orders/{id}/cancel` | PUT | P0 |
| W5.2.5 | 确认还车 | `/api/orders/{id}/return` | PUT | P1 |

**文件**：
- `backend/src/main/java/com/gybike/controller/OrderController.java`
- `backend/src/main/java/com/gybike/service/OrderService.java`
- `backend/src/main/java/com/gybike/service/impl/OrderServiceImpl.java`
- `backend/src/main/java/com/gybike/dto/OrderRequestDTO.java`
- `backend/src/main/java/com/gybike/dto/OrderResponseDTO.java`

**关键业务逻辑**：
1. **下单校验**：车辆库存检查、时间冲突检测、押金金额校验
2. **订单号生成规则**：GY + 时间戳 + 6位随机数（如 GY20260406123456）
3. **金额计算**：租金 = 单价 × 天数；总金额 = 租金 + 服务费 - 优惠折扣
4. **状态流转**：待支付(PENDING) → 已支付(PAID) → 租用中(RENTING) → 已完成(COMPLETED)/已取消(CANCELLED)

#### 3.1.1 后端支付服务

| 编号 | 任务 | 文件 | 说明 |
|------|------|------|------|
| W6.1.1 | 支付服务接口 | `service/PaymentService.java` | 支付抽象接口 |

| W6.1.5 | 支付记录DAO | `dao/PaymentRecordDAO.java` | 支付流水记录 |

**关键接口（参考后端接口文档第六章）**：

| 接口 | URL | 方法 | 说明 |
|------|-----|------|------|
| 发起支付 | `/api/payments/create` | POST | 创建微信支付预订单 |
| 支付回调 | `/api/wxpay/notify` | POST | 微信异步通知（需验签） |
| 查询支付状态 | `/api/payments/{orderId}/status` | GET | 主动查询支付结果 |
| 申请退款 | `/api/refunds` | POST | 发起退款请求 |
| 退款回调 | `/api/wxpay/refund/notify` | POST | 退款结果通知 |

**微信支付集成要点**：
- 使用微信支付API v3版本
- 需要配置：appId、mchId、apiKeyV3、serialNo（证书序列号）
- 回调通知必须验签（RSA-SHA256）
- 支付金额单位为**分**

#### 3.1.2 小程序端支付

| 编号 | 任务 | 文件 | 说明 |
|------|------|------|------|

**小程序支付调用流程**：
```javascript
// 1. 后端创建预订单 → 返回 prepay_id 等参数
// 2. 小程序调起支付
wx.requestPayment({
  timeStamp: '',
  nonceStr: '',
  package: 'prepay_id=xxx',
  signType: 'RSA',
  paySign: '',
  success: res => { /* 更新订单状态为PAID */ },
  fail: err => { /* 处理支付失败 */ }
})
```

## 六、任务优先级与风险标注

### 6.1 优先级定义

| 优先级 | 含义 | 标准 |
|--------|------|------|
| **P0** | 必须完成 | 核心业务闭环不可或缺 |
| **P1** | 应该完成 | 影响用户体验的重要功能 |
| **P2** | 可以延后 | 锦上添花的功能 |

### 6.2 关键路径（决定整体进度的任务链）

```
W5.1 数据层 → W5.2 订单API → W5.3 下单页面 → W6.1 支付模块 → W5.4 订单管理
                                                    ↓
                                          W8.1 全流程联调
```

**关键路径上的任务不可延误**，否则影响最终交付。

### 6.3 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 微信支付资质审核慢 | 支付功能无法按时开发 | 中 | 先用沙箱环境开发，并行准备资质 |
| 第三方地图接口配额不足 | 门店定位不准 | 低 | 备选腾讯地图SDK |
| 小程序审核被拒 | 上线延期 | 中 | 提前熟悉审核规范，预留一周缓冲 |
| 需求变更导致返工 | 进度延误 | 中 | 每周五做阶段性演示确认 |
