**Entity 设计**：
```typescript
class ViolationDeposit {
  id; orderId; userId; amount;
  status: 'FROZEN' | 'DEDUCTED' | 'REFUNDED' | 'PARTIAL_REFUND';
  frozenTransactionId;        // 支付宝预授权流水
  deductionAmount; refundAmount;
  violationDetail: string;    // 违章详情JSON
  deductedAt; refundedAt;
  observationEndAt;           // 30天观察期截止
  remark;
}
```

**业务规则**：
- 还车时调用支付宝预授权冻结违章押金（默认500元）
- 30天观察期自动检查 → 无违章则解冻退还
- 商家可手动发起扣除（上传违章凭证）
- 支持部分退还（扣除部分，剩余退回）
- 所有操作写入 payment 流水

```
待受理(PENDING) → 处理中(PROCESSING) → 已解决(RESOLVED) → 已关闭(CLOSED)
     ↓              ↓                           ↓
   已拒绝(REJECTED)  等待用户回复(WAITING)    重新打开(REOPENED)
                                                    ↓
                                              PROCESSING
```

**升级机制（4级超时自动升级）**：
| 级别 | 条件 | 升级到 |
|------|------|--------|
| L1→L2 | 处理中超过SLA响应时限未回复 | 主管 |
| L2→L3 | 超过解决时限×2未关闭 | 经理 |
| L3→L4 | 超过解决时限×3未关闭 | 负责人 |
| L4 | 超过48h未关闭 | 标记异常+短信通知负责人 |

**脱敏规则**：

| 字段类型 | 原始值 | 脱敏后 | 方法名 |

3. 日志输出前自动脱敏

#### T3W14-5：接口防刷限流 【P0】⭐

**现状**：common 中应有 RedisService（含Lua锁），在其基础上扩展限流

**新建/扩充**：
```
server/src/common/guards/throttle.guard.ts    # [新建/扩展现有限流]
server/src/common/decorators/throttle.decorator.ts # [新建] 限流装饰器
```

**限流策略**：

| 级别 | 限制 | 应用场景 | 配置示例 |
|------|------|---------|---------|
| IP级别 | 100次/分钟 | 全局保护 | @Throttle({window:60, limit:100}) |
| 用户级别 | 30次/分钟 | 已认证接口 | @ThrottleUser({window:60, limit:30}) |
| 接口粒度 | 自定义 | 敏感操作 | @Throttle({window:300, limit:5}, key:'login') |
| 登录接口 | 5次/15分钟 | 防暴力破解 | @Throttle({window:900, limit:5}) |
| 发送验证码 | 1次/60秒 | 防短信轰炸 | @Throttle({window:60, limit:1}) |

**使用示例**：
```typescript
@Post('login')
@Throttle({ windowMs: 900_000, limit: 5 })  // 15分钟5次
async login(@Body() dto: AlipayLoginDto) { ... }
```
