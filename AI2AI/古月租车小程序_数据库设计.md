## 一、数据库设计原则

### 1.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| **数据库名** | 小写+下划线 | `gy_bike` |
| **表名** | 小写+下划线，复数形式 | `users`, `orders` |
| **字段名** | 小写+下划线 | `user_id`, `created_at` |
| **主键** | `id` 或 `{表名单数}_id` | `id`, `user_id` |
| **外键** | `fk_{表名}_{关联表}` | `fk_orders_users` |
| **索引** | `idx_{表名}_{字段}` | `idx_users_phone` |
| **唯一索引** | `uk_{表名}_{字段}` | `uk_users_alipay_id` |

### 1.2 字段类型规范

| 数据类型 | 用途 | 示例 |
|----------|------|------|
| `BIGINT` | 主键、外键、金额（分） | `id`, `user_id`, `amount` |
| `VARCHAR(n)` | 变长字符串 | `name`, `phone` |
| `TEXT` | 长文本 | `description`, `remark` |
| `DECIMAL(10,2)` | 精确金额（元） | `price`, `fee` |
| `DATETIME(3)` | 时间戳（毫秒精度） | `created_at`, `updated_at` |
| `TINYINT` | 状态、类型枚举 | `status`, `type` |
| `JSON` | 灵活结构数据 | `extra_data`, `config` |

### 1.3 通用字段约定

每张表必须包含以下审计字段：

```sql
`created_at` DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',
`updated_at` DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',
`created_by` BIGINT NULL COMMENT '创建人ID',
`updated_by` BIGINT NULL COMMENT '更新人ID',
`deleted_at` DATETIME(3) NULL COMMENT '删除时间（软删除）'
```

### 1.4 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| **主键策略** | BIGINT自增 | 简单高效，TypeORM默认支持 |
| **软删除** | `deleted_at` 字段 | 保留历史数据，支持数据恢复 |
| **金额存储** | 分为单位（BIGINT） | 避免浮点精度问题 |
| **时间精度** | DATETIME(3)毫秒 | 订单、支付等业务需要精确时间 |
| **外键约束** | 数据库层 + 应用层 | 双重保障，但允许应用层关闭外键检查 |

## 四、索引设计说明

### 4.1 索引设计原则

1. **主键索引**：所有表使用 BIGINT 自增主键
2. **唯一索引**：业务唯一字段（如订单号、车牌号、手机号）
3. **普通索引**：高频查询字段（如状态、时间、外键）
4. **联合索引**：组合查询条件（如 user_id + status）
5. **前缀索引**：长字符串字段（如地址、描述）

### 4.2 关键索引清单

| 表名 | 索引名 | 索引字段 | 类型 | 用途 |

| `users` | `idx_users_level_id` | `level_id` | NORMAL | 按会员等级查询 |
| `orders` | `uk_orders_order_no` | `order_no` | UNIQUE | 订单号唯一 |

## 五、外键约束说明

### 5.1 外键策略

本项目采用**数据库外键约束 + 应用层校验**双重保障：

- 数据库层：定义外键约束，保证引用完整性
- 应用层：在业务逻辑中校验关联关系

### 5.2 外键约束清单

| 外键名 | 从表 | 从表字段 | 主表 | 主表字段 | 删除策略 | 更新策略 |
|--------|------|----------|------|----------|----------|----------|
| `fk_driver_licenses_users` | `driver_licenses` | `user_id` | `users` | `id` | RESTRICT | CASCADE |
| `fk_vehicles_vehicle_models` | `vehicles` | `model_id` | `vehicle_models` | `id` | RESTRICT | CASCADE |
| `fk_vehicles_stores` | `vehicles` | `store_id` | `stores` | `id` | RESTRICT | CASCADE |
| `fk_orders_users` | `orders` | `user_id` | `users` | `id` | RESTRICT | CASCADE |
| `fk_orders_vehicles` | `orders` | `vehicle_id` | `vehicles` | `id` | RESTRICT | CASCADE |
| `fk_payments_orders` | `payments` | `order_id` | `orders` | `id` | RESTRICT | CASCADE |
| `fk_user_coupons_coupons` | `user_coupons` | `coupon_id` | `coupons` | `id` | RESTRICT | CASCADE |
| `fk_tickets_users` | `tickets` | `user_id` | `users` | `id` | RESTRICT | CASCADE |
| `fk_role_permissions_roles` | `role_permissions` | `role_id` | `roles` | `id` | CASCADE | CASCADE |
| `fk_staff_roles_staff` | `staff_roles` | `staff_id` | `staff` | `id` | CASCADE | CASCADE |

### 5.3 删除策略说明

- **RESTRICT**：禁止删除有引用记录的主表数据（如用户有订单时不能删除用户）
- **CASCADE**：级联删除（如删除角色时自动删除角色权限关联）
- **SET NULL**：设置为空（如删除订单时，用户优惠券的订单引用设为空）
