### 第19周：性能优化

#### T5.19.1 前端性能优化
| 任务 | 文件/配置 | 说明 |
|------|----------|------|
| 分包加载 | `pages.json` | 将非首屏页面拆分到subPackages |
| 图片懒加载 | `components/` | 车辆卡片、评价图片懒加载 |
| 图片压缩 | 工具函数 | 上传前压缩 + WebP格式转换 |
| 组件按需加载 | 全局 | 动态import非必要组件 |
| 缓存策略 | `api/request.ts` | 接口缓存 + 缓存失效机制 |
| 骨架屏 | `components/` | 列表页骨架屏加载态 |

#### T5.19.2 后端API性能优化
| 任务 | 文件 | 说明 |
|------|------|------|
| 慢查询优化 | `*.service.ts` | EXPLAIN分析 + 索引优化 |
| 热点数据缓存 | Redis | 车辆列表/用户信息/价格计算 |
| N+1问题修复 | `*.service.ts` | 使用leftJoinAndSelect |
| 批量查询优化 | `vehicle.service.ts` | 车辆列表一次性加载关联数据 |
| 响应压缩 | `main.ts` | compression中间件 |

#### T5.19.3 数据库索引优化
| 表 | 索引 | 说明 |
|------|------|------|
| `order` | `idx_user_status` | (userId, status) 复合索引 |
| `order` | `idx_created_at` | (createdAt) 时间范围查询 |
| `vehicle` | `idx_store_status` | (storeId, status) 复合索引 |
| `payment` | `idx_order_id` | (orderId) 订单支付查询 |
| `user_coupon` | `idx_user_status` | (userId, status) 用户优惠券 |

#### T5.19.4 首页加载优化
| 指标 | 目标 | 优化措施 |

| 首屏加载 | < 2秒 | 分包 + 缓存 + 懒加载 |
| API响应 | < 500ms | 索引 + 缓存 + 压缩 |
| 包体积 | < 2MB | 分包 + 按需引入 |

#### T5.19.5 CDN配置

## 三、技术方案

### 性能优化架构
```
客户端请求 → CDN缓存 → Nginx负载 → NestJS应用 → Redis缓存 → MySQL
                 ↓           ↓            ↓
              静态资源    压缩/Gzip      热点数据缓存
```

### 监控体系
```
应用监控 → APM (响应时间/错误率/吞吐量)
数据库监控 → 慢查询/连接数/死锁
业务监控 → 订单成功率/支付成功率
告警通知 → 钉钉/短信
```

### 测试框架
| 类型 | 工具 |
|------|------|
| 单元测试 | Jest |
| E2E测试 | Cypress (商家端) |
| API测试 | Postman/Newman |
| 压力测试 | Artillery/k6 |
| 安全扫描 | OWASP ZAP |
