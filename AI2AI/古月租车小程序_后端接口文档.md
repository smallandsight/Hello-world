## 一、接口设计规范

### 1.1 请求规范

#### HTTP方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询资源 | `GET /vehicles` 查询车辆列表 |
| POST | 创建资源 | `POST /orders` 创建订单 |
| PUT | 更新资源(完整) | `PUT /vehicles/1` 更新车辆信息 |
| PATCH | 更新资源(部分) | `PATCH /orders/1/cancel` 取消订单 |
| DELETE | 删除资源 | `DELETE /coupons/1` 删除优惠券 |

#### 请求头

```
Content-Type: application/json
Authorization: Bearer {token}      # 需要认证的接口必填
X-Request-Id: {uuid}              # 请求追踪ID（可选，建议携带）
X-Platform: alipay_mini           # 平台标识: alipay_mini / admin_web
```

#### 分页参数

```
GET /vehicles?page=1&size=20&sort=created_at:desc

参数说明:
- page: 页码，从1开始，默认1
- size: 每页条数，默认20，最大100
- sort: 排序字段，格式: {field}:{asc|desc}
```

### 1.2 响应规范

#### 统一响应格式

```typescript
// 成功响应
{
  "code": 0,
  "message": "success",
  "data": { ... },            // 响应数据
  "timestamp": 1719600000000  // 时间戳(毫秒)
}

// 失败响应
{
  "code": 1001,
  "message": "参数错误",
  "data": null,
  "timestamp": 1719600000000,
  "errors": [                  // 详细错误信息（可选）
    { "field": "phone", "message": "手机号格式不正确" }
  ]
}
```

#### 分页响应格式

```typescript
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 100,           // 总记录数
    "page": 1,               // 当前页码
    "size": 20,              // 每页条数
    "pages": 5,              // 总页数
    "list": [ ... ]          // 数据列表
  },
  "timestamp": 1719600000000
}
```

### 1.3 错误码定义

| 错误码范围 | 说明 |
|-----------|------|
| 0 | 成功 |
| 1000-1999 | 通用错误（参数、格式、权限） |
| 2000-2999 | 用户模块错误 |
| 3000-3999 | 车辆模块错误 |
| 4000-4999 | 订单模块错误 |
| 5000-5999 | 支付模块错误 |
| 6000-6999 | 营销模块错误 |
| 7000-7999 | 客服模块错误 |
| 9000-9999 | 系统错误 |

#### 常用错误码

| 错误码 | 说明 |
|-------|------|
| 1001 | 参数错误 |
| 1002 | 未授权（未登录） |
| 1003 | 禁止访问（无权限） |
| 1004 | 资源不存在 |
| 1005 | 方法不允许 |
| 1006 | 请求频率超限 |
| 1007 | 请求超时 |
| 2001 | 用户不存在 |
| 2002 | 用户已存在 |
| 2003 | 认证失败 |
| 2004 | 驾驶证未认证 |
| 3001 | 车辆不存在 |
| 3002 | 车辆不可租 |
| 4001 | 订单不存在 |
| 4002 | 订单状态异常 |
| 4003 | 订单已取消 |
| 5001 | 支付失败 |
| 5002 | 退款失败 |

## 二、认证模块接口

### 2.1 用户端认证

#### 2.1.1 支付宝授权登录

```
POST /auth/alipay/login
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| authCode | string | 是 | 支付宝授权码 |
| nickName | string | 否 | 用户昵称 |
| avatar | string | 否 | 用户头像URL |

**请求示例：**

```json
{
  "authCode": "abc123def456",
  "nickName": "张三",
  "avatar": "https://..."
}
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "tokenType": "Bearer",
    "expiresIn": 604800,
    "userId": 10001,
    "isNew": false,
    "memberLevel": 2,
    "licenseStatus": 1
  },
  "timestamp": 1719600000000
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | JWT访问令牌 |
| tokenType | string | 令牌类型，固定"Bearer" |
| expiresIn | number | 过期时间（秒） |
| userId | number | 用户ID |
| isNew | boolean | 是否新用户 |
| memberLevel | number | 会员等级: 0普通1银卡2金卡3钻石 |
| licenseStatus | number | 驾驶证状态: 0未认证1已认证2审核中 |

## 十三、接口版本控制

### 13.1 版本号规则

- URL路径中包含版本号：`/v1/`, `/v2/`
- 向后兼容的修改不升级版本号
- 不兼容的修改升级主版本号

### 13.2 版本升级策略

| 变更类型 | 版本升级 | 示例 |
|---------|---------|------|
| 新增接口 | 不升级 | 新增`/v1/tickets` |
| 新增可选参数 | 不升级 | 新增可选分页参数 |
| 新增响应字段 | 不升级 | 新增返回字段 |
| 删除接口 | 升级 | 删除旧接口 |
| 修改必填参数 | 升级 | 修改请求参数 |
| 修改响应结构 | 升级 | 修改响应格式 |

### 14.2 请求签名规则（可选）

对于敏感接口（如支付），建议添加请求签名：

```
1. 将所有请求参数按字典序排序
2. 拼接成 key=value&key=value 格式
3. 在末尾追加 &app_secret={secret}
4. 对拼接字符串进行 MD5/SHA256 加密
5. 将签名值放入请求头 X-Signature
```
