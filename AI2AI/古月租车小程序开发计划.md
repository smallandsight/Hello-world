## 一、项目概述

### 1.1 项目定位

古月租车是一个专业汽车租赁服务平台，依托支付宝生态，为用户提供便捷的租车服务体验。平台连接租车用户与租赁商家，实现从车辆浏览、预订下单、支付取车到还车结算的全流程线上化。

### 1.2 核心目标

| 目标维度 | 具体指标 |
|---------|---------|
| 功能完整性 | 覆盖P0功能100%、P1功能85%、P2功能50% |
| 用户体验 | 首页加载 < 2秒，核心操作 ≤ 3步完成 |
| 系统稳定性 | API可用性 ≥ 99.5%，核心接口响应 < 500ms |
| 安全合规 | 通过支付宝小程序审核，符合《汽车租赁管理暂行规定》 |

### 1.3 产品范围

#### 用户端（7大模块）
1. **用户管理模块** — 支付宝登录、实名认证、驾驶证认证、信用免押、会员等级
2. **车辆浏览与搜索** — 首页推荐、车辆列表、详情、门店LBS定位、地图导航、收藏
3. **订单预订模块** — 在线选车、价格计算、订单确认、在线支付、取消/修改
4. **订单管理模块** — 列表/详情、取车签到、还车结算、续租申请、评价
5. **支付管理模块** — 租金支付、押金管理、违章押金、发票申请、退款
6. **评价与反馈模块** — 车辆/服务评价、用户反馈、评价展示
7. **会员与营销模块** — 会员中心、积分商城、优惠券、邀请有礼、活动中心

#### 商家端（5大模块）
1. **车辆管理** — 录入/上下架/GPS定位/维保记录
2. **订单管理** — 列表/详情、取还车确认、异常处理、导出
3. **门店管理** — 门店信息、员工账号、库存查看、调配
4. **财务管理** — 收入统计、对账、支出、结算、报表导出
5. **运营管理** — 价格设置、活动配置、数据分析仪表盘、消息通知

#### 公共服务模块
- 消息推送（支付宝订阅消息 + 短信 + 小程序内消息）
- 客服工单系统（在线客服 + AI智能客服 + 工单流转）
- 数据分析报表体系
- 异常处理规范（SOP）
- RBAC权限体系

## 三、团队配置

### 3.1 推荐团队结构（1-2人模式）

由于团队规模为1-2人，采用**全栈为主、串行为主**的开发模式：

| 角色 | 人数 | 主要职责 |
|------|------|---------|
| **项目负责人/全栈工程师** | 1人 | 架构设计、后端开发(NestJS)、数据库设计、API开发、部署运维 |
| **前端工程师** | 0-1人 | 小程序开发(uni-app)、商家后台(Vue3)、UI还原、交互优化 |

> **注**：若为1人团队，则前后端均由同一人负责，需合理分配前后端时间比例（建议前:后 = 4:6）。

### 3.2 角色能力要求

#### 全栈工程师（必配）
- 精通 TypeScript / JavaScript（ES6+）
- 熟练掌握 NestJS 框架及 Node.js 生态
- 熟悉 MySQL 数据库设计与优化
- 了解 Redis 缓存策略
- 有支付宝小程序或 uni-app 开发经验优先
- 具备基本的 DevOps 能力（Docker/Nginx/CI）

#### 前端工程师（可选配）
- 精通 Vue3 组合式 API 及 Pinia 状态管理
- 熟练使用 uni-app 开发跨平台应用
- 熟悉 Element Plus 组件库
- 具备良好的 UI/UX 还原能力
- 了解支付宝小程序开发规范

### 3.3 外部协作资源

| 资源类型 | 用途 | 备注 |
|---------|------|------|
| UI设计师 | 产品视觉设计 | 可外包/使用现成模板 |
| 支付宝技术支持 | 小程序审核对接 | 官方技术群 |
| 法务顾问 | 租车协议/隐私政策合规 | 外部咨询 |
